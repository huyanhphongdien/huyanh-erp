ALuồng mua mủ bằng Chat

Partner có mủ sẵn → chat trực tiếp nhà máy → gửi phiếu chốt → factory xác nhận → Deal.

Use case: Partner đã có lô mủ (mu tạp / mủ nước / mủ cao su) cần bán ngay, không chờ nhà máy đăng nhu cầu.

📋 Bước chi tiết

  1. Mở chat room — Partner hoặc Factory khởi tạo. b2b.chat_rooms INSERT/SELECT.
  2. Tạo phiếu chốt — Partner click icon 📎 trong chat → form BookingCard.
  3. Điền phiếu — Loại mủ, số lượng (tấn), DRC (%), giá (VNĐ/kg), đơn vị (wet/dry), ngày giao, vùng nguyên liệu, 🆕 Giao tại nhà máy.
  4. Gửi phiếu — Message message_type='booking' với metadata đầy đủ.
  5. Factory nhận — Realtime < 2s. BookingCard hiện trong chat + notification bell +1.
  6. Xác nhận — Factory click nút Xác nhận → Modal "Tạo Deal" mở, form prefill từ booking.
  7. Optional tạm ứng — Bật toggle "Tạm ứng ngay", điền số tiền + phương thức (cash/bank).
  8. Tạo Deal — Click Tạo Deal → INSERT b2b.deals status=processing.
  9. Auto cascade — Booking status='accepted', chat card "Deal đã tạo", partner notification.

BLuồng mua theo Nhu cầu

❌ Deadline phải sau ngày đăng
CHECK deadline ≥ published_at

CDeal Lifecycle — 5 Stages Chính

Sau khi Deal tạo (từ A hoặc B), đi qua 5 giai đoạn cộng tác xuyên bộ phận.

stateDiagram-v2 [*] --> processing : Deal created processing --> processing : QC đo DRC mẫu processing --> accepted : BGĐ Duyệt accepted --> accepted : Cân IN + Stock-in accepted --> accepted : Advance + Acknowledge accepted --> accepted : Settlement confirmed accepted --> settled : Settlement PAID settled --> [*] : Complete processing --> cancelled : Hủy accepted --> cancelled : Hủy (nếu chưa nhập kho) note right of processing 🔬 QC phụ trách Chưa unlock cân end note note right of accepted 👔 BGĐ đã duyệt Unlock: cân + stock-in + advance end note note left of settled 💰 Tier auto upgrade ledger running_balance = 0 end note

📍 Stage × Bộ phận

Stage UI DB status Bộ phận Hành động Output
🔹 Chốt processing Auto Deal mới tạo, chờ QC chat card "Deal created"
🔹 QC processing 🔬 QC Đo DRC mẫu, update actual_drc DRC actual > 0
🔹 Duyệt accepted 👔 BGĐ Review + click "Duyệt Deal" Unlock cân + stock-in + advance
🔹 Nhập kho accepted 🚚 Cân + Kho Xe cân IN → stock-in KHO-NVL stock_batches + stock_in_count++
🔹 Tạm ứng accepted 💰 Kế toán Tạo advance + partner acknowledge partner_ledger entry (advance_paid)
🔹 Quyết toán accepted 💰 Kế toán Settlement items (gross - advance) settlement_receivable ledger
🔹 Thanh toán settled 💰 Kế toán Payment paid → cascade settled ledger = 0 · tier auto upgrade

🔒 Sprint J Guards (CRITICAL)

⚖️
Cân xe IN
Trigger trg_enforce_weighbridge_accepted reject nếu deal ≠ accepted/settled.
📦
Stock-in B2B
Trigger trg_enforce_b2b_stock_in_accepted reject nếu source_type='b2b' và deal chưa accepted.
💸
Tạm ứng
App-layer + DB check: advance chỉ tạo được khi deal.status = accepted.
