VARIANT A

ECOUNT Pro — branded green

Inspiration: 2 ảnh ECOUNT bạn paste + TabbedWorkspace.tsx hiện tại + airtable.md (dense table)

Giữ nguyên ngôn ngữ Vietnamese ERP: sidebar xanh đậm, tab card cream gradient, sub-tab row cho detail page con. Đây là cách nâng cấp tăng dần cái đang có ở Kho — risk thấp nhất, phù hợp cho rollout sang Sales/HR/Production.

Ưu điểm
  • Reuse được TabbedWorkspace.tsx + tabStore
  • User Việt quen layout này (giống MISA/FAST/ECOUNT)
  • Density cao — vừa nhiều thông tin trên 1 màn hình
  • Dễ rollout từng module 1, không cần đập đi xây lại
Hạn chế
  • Dày đặc cho user mới — cần training
  • Mobile khó dùng — bắt buộc fallback navigate
  • Không có command palette (Ctrl+K) → search chậm

VARIANT B

Linear Dark Dense — software-craft

Inspiration: linear.app.md — near-black canvas, lavender accent #5e6ad2, IDE-like density

Theme tối toàn bộ. Rail icon-only bên trái + sidebar ngữ cảnh + tab strip cao 36px. Phong cách như IDE / VS Code. Đẹp, hiện đại, nhưng không phù hợp văn phòng kế toán Việt Nam — user lớn tuổi sẽ phản đối màn hình tối. Để tham khảo cho dashboard quản trị / Coverage Dashboard.

Ưu điểm
  • Density cực cao, đẹp với dữ liệu nhiều
  • Phù hợp cho admin/dev/BGĐ xem dashboard
  • Tab strip 36px tiết kiệm chiều dọc
Hạn chế
  • User văn phòng VN không quen dark theme
  • Số liệu kế toán in giấy → chỉ dùng được cho web
  • Đập hoàn toàn cái đang có — rủi ro cao

VARIANT C

Notion Workspace — friendly editorial

Inspiration: notion.md, claude.md — cream canvas, serif headline, pill tabs

Sidebar Notion-style với search trên cùng + pinned items + nested. Tab dạng pill (bo tròn) thay vì card. Headline serif (Cormorant) cho cảm giác editorial. Phù hợp cho dashboard B2B hoặc Settings, kém phù hợp cho nhập liệu nhanh.

Ưu điểm
  • Cảm giác hiện đại, ít doạ user mới
  • Sidebar search ngay đầu — tìm module nhanh
  • Pill tab nhẹ nhàng, không cứng nhắc
Hạn chế
  • Mất tính enterprise-Vietnamese (kế toán không quen)
  • Density thấp hơn — ít data trên 1 màn hình
  • Pill tab khó nhìn dirty indicator

VARIANT D

Stripe Master/Detail — financial clean

Inspiration: stripe.md — 3-pane (sidebar/list/detail), tab nằm trong detail pane, accent #635bff

Layout 3 cột: nav module / list đơn / detail tabs. Tab chỉ tồn tại trong cột detail. Không multi-tab toàn cục — mỗi lần chọn 1 đơn sẽ mở detail của đơn đó với các tab nội bộ (info / items / payment / log). Phù hợp cho Settlements, Phải thu, Phải trả.

Ưu điểm
  • List + detail luôn nhìn thấy đồng thời
  • Số liệu tài chính hiển thị rất rõ
  • Density vừa phải — không quá tải
Hạn chế
  • Không multi-tab thật — chỉ 1 detail tại 1 thời điểm
  • Cần 3 cột → màn hình < 1280px chật
  • Không tái sử dụng được TabbedWorkspace hiện có

VARIANT E

Hybrid — ECOUNT × Linear × hiện trạng

Tổng hợp: ECOUNT layout + Linear icon rail + command palette + TabbedWorkspace.tsx + Anthropic warm cream highlights

Phương án thực tế nhất để rollout toàn ERP. Giữ branded green + cream tab card hiện tại, thêm 3 thứ: (1) Icon rail 56px bên ngoài cùng để switch giữa các module mà không mất sidebar context; (2) Command palette Ctrl+K ở topbar để tìm/mở bất kỳ entity nào; (3) Stat card cream trên đầu mỗi page để hiện KPI tổng quan. Toàn bộ logic TabbedWorkspace.tsx + tabStore + sub-tab row giữ nguyên.

Ưu điểm
  • Reuse 100% code TabbedWorkspace hiện có
  • Module switch nhanh qua icon rail (không cần mở menu)
  • Ctrl+K mở entity bất kỳ — đáp ứng request "tab đa tác vụ"
  • Header stat cream (Anthropic-warm) → enterprise vẫn cảm giác mềm
  • Dễ rollout dần: Sales → HR → Production → Accounting → B2B
Hạn chế
  • Cần thêm component CommandPalette + module rail
  • Tốn thêm 56px chiều ngang — màn hình < 1280px hơi chật

Khuyến nghị: chọn Variant E (Hybrid)

Lý do — bạn đã có TabbedWorkspace chạy ổn ở Kho. Việc cần làm là rollout pattern đó sang các module khác, KHÔNG phải đập đi xây lại.

  1. Reuse 95% code hiện có — TabbedWorkspace.tsx, tabStore, sub-tab row đều giữ nguyên.
  2. Thêm 2 component mới: ModuleRail.tsx (icon rail 56px) + CommandPalette.tsx (Ctrl+K). Cả 2 đều standalone, không đụng vào logic tab.
  3. Thêm 1 layout: StatHeader.tsx để hiển thị 4 KPI card cream phía trên mỗi list page (đáp ứng yêu cầu "tab đa tác vụ" với context hữu ích).
  4. Rollout từng module: Sales (next, vì user đang làm SO) → HR → Production → Accounting → B2B Portal cuối cùng.
  5. Variant B & C để dành: Coverage Dashboard có thể dùng style Linear (dark) như 1 page riêng. B2B Portal có thể dùng pill tab style Notion.

Roadmap rollout (nếu chọn Variant E)

Tuần 1
Build ModuleRail + CommandPalette Component src/components/layout/ModuleRail.tsx + CommandPalette.tsx. Test trên Sales trước.
Tuần 1
Build StatHeader Component src/components/common/StatHeader.tsx nhận props 4 KPI. Reuse cho mọi list page.
Tuần 2
Rollout Sales Wrap pages bán hàng (orders, customers, contracts) vào TabbedWorkspace. Đăng ký detail components vào tabStore registry. UAT với BGĐ.
Tuần 3
Rollout HR + Production Áp dụng cho attendance, payroll, lịch sản xuất, QA. Test với phòng nhân sự + xưởng.
Tuần 4
Rollout Accounting + B2B Settlements, Phải thu, Phải trả, B2B Portal. Lưu ý: B2B portal có thể chọn pill tab Notion-style nếu BGĐ thấy hợp.
Tuần 5
Coverage Dashboard với Linear theme Trang riêng cho BGĐ xem KPI tổng. Dark theme + dense table. Truy cập từ rail icon "📊".
